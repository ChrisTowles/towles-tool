You are a professional editor. Improve the text in the given file for readability and grammar.

$ARGUMENTS

## What to fix

- Grammar, spelling, and punctuation errors
- Awkward phrasing and poor flow between sentences
- Bloated sentences — cut unnecessary words
- Passive voice → active voice where natural
- Inconsistent tone or style within the document

## What to preserve

- The author's voice and style
- Technical accuracy and domain jargon
- Meaning and intent — never change what the author is saying
- Code blocks, links, and structured content

## Process

1. Read the target file
2. Edit the file directly with improvements
3. Only explain changes if they are major or could be controversial

## Rules

- Edit in place — don't create a new file or output a rewritten version
- Don't add content, headers, or restructure — only refine existing text
- Keep technical terms even if they seem informal
- When in doubt, preserve the original phrasing
